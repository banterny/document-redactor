# document-redactor v1.0.0

_Released 2026-04-10_

## What this is

A one-file, offline, in-browser DOCX redactor built for lawyers who want to hand their documents to ChatGPT / Claude / any other LLM — but can't, because the file has company names, counterparty details, phone numbers, 주민등록번호, account numbers, or case numbers that must not leave the machine.

Drop the `.docx` onto the tool. Review what it found. Click Apply. Download the redacted copy. No internet connection, no installation, no terminal, no account, no upload. One HTML file, ~180 KB, runs in your browser from your disk.

## What's in v1.0.0

- **Bilingual detection** — Korean + English PII regex sweep (주민등록번호, 사업자등록번호, EIN, phones, emails, bank accounts, credit cards) + user-provided entity seeds with automatic variant expansion.
- **Defined-term handling (D9)** — the tool parses `"X" means Y` and Korean `("X"이라 함은)` patterns and classifies aliases as literal or defined. Role labels like "the Buyer" are kept unredacted by default so downstream readers can still parse the contract structure.
- **Cross-run redaction** — Word splits text across runs like `<w:t>ABC Corpo</w:t><w:t>ration</w:t>`; the tool's coalescer reassembles a logical text view, matches, and surgically rewrites only the affected runs. No silent leaks from naive string replace.
- **Full DOCX scope coverage** — body, footnotes, endnotes, comments, headers, footers. Track changes are flattened, comments are stripped, and document metadata (author, last modified by, title, etc.) is scrubbed.
- **Byte-stable output** — same input + same selections produce a byte-identical `.docx` with a matching SHA-256. Two reviewers hashing the same output get the same digest, which makes the Kakao-message distribution story actually work.
- **Four-layer "no network" enforcement** — ESLint ban at source, build-time token scan, runtime `connect-src 'none'` CSP, and a SHA-256 sidecar for download integrity.
- **English + Korean documentation** — [README.md](../README.md), [README.ko.md](../README.ko.md), [USAGE.md](../USAGE.md), [USAGE.ko.md](../USAGE.ko.md).

## Verify your download (recommended)

Save both files to the same directory and run:

```bash
sha256sum -c document-redactor.html.sha256
# expected output:
#   document-redactor.html: OK
```

If you see `OK`, the file is byte-identical to what the author published. Anything else — do not run it.

## Known limitations

- **DOCX only.** PDF input is planned for a future update — for now, convert PDFs to DOCX first (Word, Google Docs, or your PDF tool's export feature).
- **Level picker is cosmetic.** Only the Standard rule set runs in v1; Conservative and Paranoid are UI stubs and will ship in v1.1.
- **No click-to-select in the document preview.** The preview pane is a placeholder; all candidate review happens through the right-hand panel.
- **View source button is disabled.** Planned for v1.1 — the self-hash modal will compare the running file against the published release hash. The Audit log button from early mocks has been removed from the roadmap — this tool is designed for incognito, one-shot use where leaving no state is a feature.
- **Layout degrades to 2-column below 720 px.** The 3-column desktop layout requires ≥1024 px for comfortable use.
- **No OCR.** If your DOCX contains images of text (scanned PDFs imported to Word), the text inside those images is not processed. OCR is not on the roadmap — browser-based OCR engines would break the single-file distribution model. Use a separate OCR tool first.
- **English + Korean contracts tested end-to-end.** Other document types (opinions, briefs, memos) work — the engine is text-based, not contract-specific — but v1's test fixture is a bilingual NDA.
