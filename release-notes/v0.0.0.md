# document-redactor v0.0.0 (dev-preview)

_Released 2026-04-10 — internal pipeline test only_

## What this is

This is a **dev-preview release** created purely to verify that the
`.github/workflows/release.yml` pipeline works end-to-end:

1. Tag push (`v0.0.0`) triggers the workflow
2. Tag regex guard accepts the tag
3. Checkout at the tag ref succeeds
4. `actionlint` passes
5. bun install + lint + typecheck + test (422 tests) + build all pass
6. `shipGate()` writes the SHA-256 sidecar next to the HTML
7. `sha256sum -c` verifies sidecar integrity
8. `gh release create` publishes both files as assets

If you can read this note on a live GitHub Release, every step above worked.

## What this is NOT

- **Not v1.0.0.** v1 is not yet ready — still needs the README (F-3), the
  pre-public transition (filter-repo + secret scan + visibility flip), and
  an author decision to ship.
- **Not for distribution.** This release will be deleted from GitHub after
  the pipeline verification completes. Do not share the assets attached
  here.
- **Not a functional guarantee.** The tool itself (`document-redactor.html`)
  is fully built and works end-to-end — this test just proves the release
  automation layer works. The first official user-facing release is v1.0.0.

## Verify your download (recommended)

Save both files to the same directory and run:

```bash
sha256sum -c document-redactor.html.sha256
# expected output:
#   document-redactor.html: OK
```

## Cleanup plan

After the release.yml run succeeds, the author will:

1. `gh release delete v0.0.0 --yes`
2. `git push --delete origin v0.0.0`
3. `git tag -d v0.0.0`
4. Remove `release-notes/v0.0.0.md`
5. Commit the cleanup
6. Push to main

The workflow run history in the Actions tab is permanent (GitHub does not
allow deleting workflow runs via API for audit reasons), but the tag, the
release, and this file will all disappear.
